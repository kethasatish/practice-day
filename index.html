Execution Context and Call Stack

An execution context is an abstract concept describing the environment within which JavaScript code is executed. Every time a function is invoked, a new execution context is created and pushed onto the call stack.

Global Execution Context: Created when the script first runs. It contains the global object (window in browsers), this, and any globally scoped variables or functions. Function Execution Context: Created whenever a function is invoked. It has its own this, arguments, and local variables.

Call Stack: JavaScript uses a call stack to manage execution contexts. When a function is called, a new execution context is pushed onto the stack. When the function returns, the execution context is popped off the stack. javascript

10.JASON API(114-126 days).

It seems like you're referring to JSON (JavaScript Object Notation),

which is a lightweight data interchange format commonly used for transmitting data between a server and a client over the web.

JSON is easy for humans to read and write, and it's also easy for machines to parse and generate.

An API (Application Programming Interface) that uses JSON for data exchange is often called a JSON API.

JSON APIs are popular because JSON can represent structured data such as arrays and objects,

making it versatile for transmitting various types of information.

JSON APIs are widely used in web development, particularly in RESTful APIs (Representational State Transfer).

In a RESTful API, HTTP methods such as GET, POST, PUT, DELETE, etc., are used to perform CRUD (Create, Read, Update, Delete) operations on resources, and JSON is typically used to represent the data being sent or received in the HTTP requests and responses.
